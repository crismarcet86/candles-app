1. Descripción General del Sistema

Candles App es un sistema web de gestión orientado a emprendimientos de velas artesanales. Cubre el ciclo completo del negocio: desde la gestión de ingredientes y stock, pasando por la calculadora de costos, hasta la emisión de cotizaciones (proformas) y la confirmación de pedidos con descuento automático de inventario.

1.1 Objetivos del Sistema

  • Gestionar el catálogo de ingredientes (cera, esencia, pabilos, colorantes) con sus unidades de medida y stock.
  • Administrar moldes y tipos de molde con cálculo automático de gramos de cera.
  • Calcular el costo de fabricación por vela usando ingredientes, % de fragancia, color y mano de obra.
  • Emitir proformas/cotizaciones a clientes con descuentos y exportación en PDF.
  • Confirmar pedidos descontando stock automáticamente con conversión de unidades.
  • Generar reportes gerenciales: ventas por período, stock bajo, top clientes, KPIs.
  • Gestionar la configuración del negocio (nombre, RUC, logo) que aparece en todos los PDFs.

1.2 Roles de Usuario

RolAcceso
adminAcceso completo: CRUD de usuarios, configuración del negocio, todas las funcionalidades.
userAcceso operativo: gestión de inventario, proformas, calculadora y reportes. Sin gestión de usuarios ni configuración.

El sistema sigue una arquitectura de cliente-servidor de dos capas con separación completa entre frontend y backend:

2.3 Flujo de Autenticación

[Login] POST /api/auth/login { username, password }
    │
    ▼
User.findByUsername(username) → bcrypt.compare(password, hash)
    │
    ▼
jwt.sign({ id, username, role, name }, JWT_SECRET, expires: 8h)
    │
    ▼
Respuesta: { user, token }
    │
    ▼
Frontend guarda token en localStorage
    │
    ▼
HTTP Interceptor inyecta: Authorization: Bearer <token>
    │
    ▼
requireAuth: jwt.verify(token) → req.user = payload

8. Autenticación y Seguridad

8.1 JWT

  • Se genera con jsonwebtoken al hacer login o register.
  • El payload contiene: { id, username, role, name }
  • Expiración configurable vía JWT_EXPIRES (default: 8h)
  • El frontend lo almacena en localStorage bajo la clave candles_token
  • Un interceptor HTTP Angular lo inyecta automáticamente en cada request como Authorization: Bearer <token>

8.2 Contraseñas

  • Se hashean con bcryptjs con salt de 10 rondas antes de guardar en DB.
  • La comparación se hace con bcrypt.compare(plain, hash) — nunca se almacena la contraseña en texto claro.
  • El cambio de contraseña requiere el token activo y la contraseña actual como verificación.

8.3 Middlewares de Seguridad

MiddlewareFunción
requireAuthVerifica el token JWT. Si es inválido o expirado → 401. Carga el payload en req.user.
requireAdminVerifica que req.user.role === 'admin'. Si no → 403. Debe ir después de requireAuth.
validateEjecuta validationResult(req) de express-validator. Si hay errores → 400 con detalle.
errorHandlerMiddleware final: captura cualquier error no manejado → JSON { ok: false, message }.

8.4 Guards del Frontend

GuardComportamiento
AuthGuardProtege rutas de /dashboard. Redirige a /auth/login si no hay token.
NoAuthGuardProtege rutas de /auth. Redirige a /dashboard si ya hay token activo.
AdminGuardRestringe rutas a usuarios con role === 'admin'. Redirige al dashboard si no cumple.

8.5 CORS

El backend acepta requests solo del origen configurado en CORS_ORIGIN (típicamente http://localhost:4200 o la IP LAN del servidor). Configurable en src/app.js.

11. Lógica de Negocio Crítica

11.1 Confirmación de Proforma — Transacción Central

Método: Proforma.confirm(id) en src/models/Proforma.js

Este método es la operación más crítica del sistema. Se ejecuta en una sola transacción ACID:

BEGIN TRANSACTION
│
├── 1. Cargar proforma con ítems (findById)
│      └── Verifica que status === 'borrador'
│
├── 2. Verificar stock para ítems directos (product_id no nulo)
│      ├── FOR UPDATE (bloqueo pesimista)
│      └── gramsToNativeUnit(quantity, unit_abbr) → compara con stock
│
├── 3. Verificar stock para ítems de preset (preset_id no nulo)
│      ├── Cargar calculation_preset_items
│      ├── FOR UPDATE en cada producto
│      └── delta = gramsToNativeUnit(grams, unit_abbr) × quantity
│
├── 4. Marcar proforma como 'confirmada'
│
├── 5. Crear registro en orders (copia de subtotal/discount/total)
│
├── 6. Copiar proforma_items → order_items
│      └── Descontar stock de ítems directos (con conversión)
│
├── 7. Descontar stock de ingredientes de presets
│      └── UPDATE products SET stock = stock - delta
│
└── 8. Marcar presets usados como is_active = 0
         (ya fueron consumidos en esta orden)

COMMIT (o ROLLBACK si cualquier paso falla)

11.2 Conversión de Unidades

Los gramos en calculation_preset_items siempre son gramos reales. Al descontar del stock (que está en la unidad nativa del producto), se aplica:

function gramsToNativeUnit(grams, unitAbbr) {
  switch (unitAbbr.toLowerCase()) {
    case 'kg': return grams / 1000;
    case 'lb': return grams / 453.592;
    case 'oz': return grams / 28.3495;
    default:   return grams; // g, ml, u, l, etc.
  }
}

Esta función se aplica tanto en la verificación de stock como en el descuento efectivo.

11.3 Fórmula de Cera en Moldes

wax_grams = total_grams × 0.90 × 1.05
  • total_grams: peso del agua al llenar el molde (método de desplazamiento)
  • × 0.90: densidad de la cera ≈ 90% del agua
  • × 1.05: margen del 5% para compensar contracción al enfriarse

11.4 Cálculo de Costo en la Calculadora

Para cada línea de ingrediente:
  - En kg:  costoPorGramo = price / 1000;  lineaCosto = grams × costoPorGramo
  - En ml:  costoPorMl   = price;          lineaCosto = ml × costoPorMl
  - En u:   costoPorUnd  = price;          lineaCosto = cantidad × costoPorUnd
  - En g:   costoPorGramo = price;         lineaCosto = grams × costoPorGramo

% Fragancia (solo si is_fragrance = 1):
  fragranceMl = wax_grams × (pct / 100)
  cera reducida en fragranceMl (misma densidad)

Costo total por vela:
  totalCosto = ΣlineaCostos
             + (includesColor ? 0.10 : 0)
             + (laborCost × laborHours)

Margen:
  ganancia = sellPrice - totalCosto
  margen % = (ganancia / sellPrice) × 100

11.5 Visibilidad de Mano de Obra en Proforma

La mano de obra (labor_cost) se absorbe silenciosamente en el subtotal mostrado al cliente. En el PDF y en la vista de detalle:

displaySubtotal = proforma.subtotal + proforma.labor_cost
total mostrado  = displaySubtotal - descuento

La línea "Mano de obra" no aparece en el desglose de ítems al cliente.

15.4 Primeros Pasos en el Sistema

  1. Configurar el negocio: en Configuración (admin), ingresar nombre, RUC y subir logo. Este aparece en todos los PDFs.
  2. Crear categorías: marcar is_fragrance = 1 para categorías de esencias/fragancias.
  3. Crear unidades: g, kg, ml, oz, lb, u (o las que use el negocio).
  4. Crear ingredientes: asignar categoría, unidad, precio y stock inicial.
  5. Crear tipos de molde y moldes: ingresar el peso total de agua para calcular automáticamente los gramos de cera.
  6. Usar la calculadora: seleccionar molde, agregar ingredientes, definir % de fragancia, mano de obra y precio de venta. Guardar como preset.
  7. Crear cliente y proforma: agregar ítems libres o desde un preset. Descargar PDF para enviar al cliente.
  8. Confirmar pedido: en el detalle de proforma, confirmar para generar la orden y descontar el stock automáticamente.
